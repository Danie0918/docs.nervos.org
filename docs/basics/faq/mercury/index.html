Mercury FAQ

Q: How to use Mercury RPC? What are RPCâ€™s access paths?â€‹

A: First, requesting Mercury RPC requires a running Mercury node. The default RPC URI of a local Mercury node ishttp://127.0.0.1:8114. If needed, you can modify it in the Mercury configuration file. Mercury pulls data from CKB nodes and provides APIs for CKBytes transfers, UDT transfers, etc.

The community also provide the following public nodes:

  • Testnet URL: https://mercury-testnet.ckbapp.dev/
  • Mainnet URL: https://mercury-mainnet.ckbapp.dev/

Public nodes are suitable only for quick tests. For better stability and performance, it is recommended to use your own nodes instead of public ones.

If you want to call Mercury RPC via SDK, these are available:ckb-sdk-java, ckb-sdk-go.

Q: What are the hardware requirements for running Mercury?â€‹

A: Recommended hardware is 4-core CPU, 8G RAM and 50G hard disk.

Q: What address types are currently supported by Mercury?â€‹

A: Mercury supports both full and short address formats. It supports the most common lock scripts as well, including Secp256, ACP, Cheque and PW-lock.

Q: How to get the complete list of transaction info from Mercury, including input and output addresses, time and amount?â€‹

A: You can use query_transactions to get transaction history in Mercury. It allows the user to query transaction history from addresses, asset types etc. For more details on the parameters, return value descriptions and call samples refer to here. You can either choose to run Mercury locally or access the public nodes provided by Mercury:

  • Mainnet URL: https://mercury-mainnet.ckbapp.dev
  • Testnet URL: https://mercury-testnet.ckbapp.dev

Also, you can call the query_transactions API by using ckb-sdk-java or ckb-sdk-go. For more information, see the ckb-sdk-java and ckb-sdk-go samples.
